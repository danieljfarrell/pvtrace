# Changelog

## [3.0.0]

### Added

- Command line interface `pvtrace-cli` with `simulate`, `show`, `count`,
  `spectrum` and `time` commands.
- YAML scene file format with JSON-schema validation, so scenes can be
  described without writing Python.
- Simulation results are exported to a SQLite database for analysis.
- Multiprocessing support built into `scene.simulate(num_rays, workers=...)`.
- `Ray.duration` tracks the optical time of flight, including the
  refractive index of the containing material, enabling time-resolved
  analysis.
- `Event.KILL` for rays terminated by the tracer (for example when
  exceeding the maximum step count) and `Event.NONRADIATIVE` for rays
  lost non-radiatively.
- New documentation based on mkdocs with CLI tutorials.

### Changed

- `photon_tracer.follow` is now implemented on top of the
  `photon_tracer.step_forward` generator, which yields
  `(ray, event, metadata)` tuples step by step.
- Default worker count for multiprocessing is half the CPU count, which
  testing showed scales better because NumPy's internal threads contend
  with worker processes (see PR #48).
- Renderer `max_histories` default reduced to keep memory bounded during
  long interactive sessions.
- Requires Python 3.10 or later. Thanks to Dario Cambie (@dcambie) for
  the original Python 3.8 and Windows compatibility work (#51).
- Updated for NumPy 2: removed deprecated aliases and stricter
  `copy=False` semantics.
- Replaced the abandoned `asciiplotlib` terminal plotting dependency
  with its successor `termplotlib`.

### Removed

- `Ray.is_alive` attribute. Termination is expressed through events
  (`Event.EXIT`, `Event.KILL`, `Event.NONRADIATIVE`, `Event.REACT`).
- `Ray.propagate` now requires the refractive index of the material so
  that the time of flight can be accumulated.

### Fixed

- Missing dependencies (`typer`, `pyyaml`, `jsonschema`, `scipy`) are now
  declared, and the database schema and scene-spec JSON schema ship with
  the package so the CLI works from an installed wheel.
- Multiprocessing pool executor bug when simulating with a results queue.

### Security

- SQL queries generated by the CLI use parameterised statements instead
  of string interpolation.

## [2.1.6]

Last release of the 2.x series, November 2020. See the GitHub releases
